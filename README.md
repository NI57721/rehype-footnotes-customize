# rehype-footnotes-customize
[![npm version](https://img.shields.io/npm/v/rehype-footnotes-customize.svg)](https://www.npmjs.com/package/rehype-footnotes-customize)

A rehype plugin that customizes footnote labels and backreference text on a per-file basis.

This plugin allows you to change the text of footnote labels and backreference links generated by rehype/remark footnotes based on the file path.

## What this plugin does

This plugin modifies the output of footnotes generated by rehype/remark by replacing:

- the footnote section label (`#footnote-label`)
- the backreference link (`data-footnote-backref`)

Rules can be applied conditionally based on the processed file path.

## Installation
```bash
# npm
npm install rehype-footnotes-customize

# pnpm
pnpm add rehype-footnotes-customize

# Yarn
yarn add rehype-footnotes-customize
````

## Usage

Examples:

```js
import rehypeFootnotesCustomize from 'rehype-footnotes-customize'

export default {
  rehypePlugins: [
    [
      rehypeFootnotesCustomize,
      [
        {
          path: /^\/src\/blog\/ja\//,
          footnoteLabel: '脚註',
          footnoteBackContent: '戻る',
        },
        {
          path: /^\/src\/blog\/de\//,
          footnoteLabel: 'Fußnoten',
          footnoteBackContent: 'Zurück',
        },
      ],
    ],
  ],
}
```

## Options
The plugin accepts an array of configuration objects.

```ts
interface RehypeFootnotesCustomizeOptions {
  path: RegExp | string
  footnoteLabel?: string
  footnoteBackContent?: string
}
```

### `path`
Type: `RegExp | string`

Matches the file path of the processed document.
If the path matches, the rule will be applied.
If `path` is a string, the rule is applied when the file path includes that string.

Examples:

```js
path: /^\/blog\/de\//,
path: "/content/german/",
```

### `footnoteLabel`

Type: `string`

Replaces the label used for the footnotes section (the element with `id="footnote-label"`).

### `footnoteBackContent`

Type: `string`

Replaces the content of the backreference link (`<a data-footnote-backref>`).
